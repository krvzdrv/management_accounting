# Повторная авторизация после добавления тестового пользователя

## Запуск скрипта

Выполните в терминале:

```bash
cd "/Volumes/02 Data/work/Alumineu/GitHub/management_accounting"
node update-scripts.js
```

## Что произойдет

1. **Авторизация через OAuth:**
   - Скрипт покажет URL для авторизации
   - Откройте URL в браузере
   - Теперь авторизация должна пройти успешно (без ошибки 403)
   - Разрешите доступ
   - Скопируйте код авторизации
   - Вставьте код в терминал

2. **Обновление файлов:**
   - Скрипт прочитает все `.gs` файлы из локальной папки
   - Отправит их в Apps Script проект через API
   - Обновит каждый файл в проекте

3. **Результат:**
   - Вы увидите сообщения о прогрессе для каждого файла
   - В конце будет сводка: сколько файлов обновлено

---

## После успешной авторизации

После ввода кода авторизации:

1. Скрипт сохранит токен в `token.json`
2. Начнет обновлять файлы в Apps Script
3. Покажет прогресс для каждого файла
4. В конце выведет сводку

---

## Если все еще ошибка 403

Если после добавления пользователя все еще ошибка:

1. Убедитесь что email правильный: **volosevich.flexy@gmail.com**
2. Подождите 1-2 минуты (изменения могут применяться не сразу)
3. Попробуйте открыть URL в режиме инкогнито
4. Убедитесь что используете правильный Google аккаунт
