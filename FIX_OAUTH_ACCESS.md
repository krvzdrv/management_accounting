# Исправление ошибки доступа OAuth

## Проблема

Ошибка: **"Доступ заблокирован: приложение не прошло проверку Google"**

Это происходит потому что OAuth Consent Screen находится в режиме "Testing" и ваш email не добавлен в список тестовых пользователей.

## Решение: Добавить тестового пользователя

### 1. Откройте OAuth Consent Screen

1. Откройте [Google Cloud Console](https://console.cloud.google.com)
2. Выберите ваш проект: **calcium-vial-378411**
3. Перейдите в **APIs & Services** → **OAuth consent screen**

### 2. Добавьте тестового пользователя

1. На странице OAuth consent screen найдите раздел **Test users** (Тестовые пользователи)
2. Нажмите **+ ADD USERS** (Добавить пользователей)
3. Введите email: **volosevich.flexy@gmail.com**
4. Нажмите **Add** (Добавить)
5. Нажмите **Save** (Сохранить)

### 3. Повторная авторизация

После добавления пользователя:

1. Вернитесь в терминал
2. Запустите скрипт снова:
   ```bash
   node update-scripts.js
   ```
3. Откройте URL авторизации в браузере
4. Теперь авторизация должна пройти успешно

---

## Альтернативное решение: Публикация приложения (не рекомендуется)

Если вы хотите чтобы любой пользователь мог авторизоваться, можно опубликовать приложение, но это требует:
- Заполнения дополнительной информации
- Возможно верификации Google (долгий процесс)
- Для личного использования не нужно

**Рекомендация:** Используйте режим "Testing" и добавляйте тестовых пользователей по мере необходимости.

## После исправления

После добавления email в тестовые пользователи:
