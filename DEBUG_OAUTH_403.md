# Отладка ошибки 403: access_denied

## Проблема

Ошибка 403 все еще появляется, даже после добавления тестового пользователя.

## Возможные причины

1. Email не был правильно добавлен в тестовые пользователи
2. Изменения еще не применились (нужно подождать)
3. Используется другой Google аккаунт
4. OAuth Consent Screen не полностью настроен

## Решение: Проверка настроек

### 1. Проверьте список тестовых пользователей

1. Откройте [Google Cloud Console](https://console.cloud.google.com)
2. Выберите проект: **calcium-vial-378411**
3. Перейдите: **APIs & Services** → **OAuth consent screen**
4. Нажмите **"EDIT APP"** (Редактировать приложение)
5. В левом меню выберите **"Test users"**
6. **Проверьте список:**
   - Должен быть email: **volosevich.flexy@gmail.com**
   - Если его нет - добавьте снова
   - Если есть - удалите и добавьте заново

### 2. Убедитесь что используете правильный аккаунт

При авторизации:
- ✅ Используйте аккаунт: **volosevich.flexy@gmail.com**
- ❌ Не используйте другие аккаунты

### 3. Подождите 1-2 минуты

После добавления/изменения тестового пользователя:
- Подождите 1-2 минуты
- Изменения могут применяться не сразу

### 4. Попробуйте в режиме инкогнито

1. Откройте браузер в режиме инкогнито (Cmd+Shift+N на Mac)
2. Откройте URL авторизации
3. Авторизуйтесь

### 5. Проверьте статус OAuth Consent Screen

На странице OAuth Consent Screen:
- Статус должен быть: **"Testing"** (Тестирование)
- Не должен быть: **"Published"** (Опубликовано) - это требует верификации

## Альтернативное решение: Проверьте настройки приложения

### Проверьте название приложения

1. На странице OAuth Consent Screen
2. Проверьте **App name** (Название приложения)
3. Должно быть: **"Management Accounting Updater"** или что-то похожее
4. Если название другое - это может быть проблема

### Проверьте email разработчика

1. На странице OAuth Consent Screen
2. Проверьте **User support email**
3. Должен быть: **volosevich.flexy@gmail.com** (ваш email)
4. Если другой - измените на ваш
